## v0.9.2 — Anti-mashing emergencies (#63)

**Problem:** you could mash the first choice on every event without reading and still finish. The choice system was decorative.

**Solution:** measure decision time per modal, score a mash tally, and pre-empt the normal event roll with a catastrophic emergency when the tally trips a threshold.

### Heuristic (`src/systems/clickMetrics.js`)

```
minReadMs:           1200     // floor below which any decision is "too fast"
readMsPerChar:       35       // scales expected time with body length
mashScoreThreshold:  3        // one fast click trips it
maxEmergenciesPerRun: 5
emergencyCooldownDelta: -2    // applied when an emergency fires
```

Five decision buckets from *didNotRead* (+3) → *thoughtful* (-1) drive the score. Accessibility: the floor + per-char rate are tunable in one place.

### Emergency content (`src/content/emergencies.js`)

Six multi-stage scenarios (12 stages total), all following the same pattern: description spells out the correct choice; two wrong options are catastrophic.

1. **Cabin breach** — locker color → scrubbers setting
2. **Reactor flux excursion** — handle direction → rod drop percentage
3. **Nav blackout / dust storm** — ridge direction → relay passphrase
4. **Airlock jammed (crew outside)** — vent method → injector order
5. **Radiation storm** — shelter side → wait duration
6. **Software lockup** — reboot key → which subsystem to disable

Each stage has **3 variants** with swappable clue words. At fire time, one variant is chosen per stage and the choice order is shuffled. A player who dies and retries sees different labels and the correct button in a different slot — forcing a real read, not pattern-matching.

### Playtest (`sim/mashPlaytest.mjs`)

1000 runs per profile, identical strategy (always pick choice #0), only decision time differs:

| Profile | Win rate | Emergencies/run | Correct-pick rate |
|---|---|---|---|
| Masher (50 ms) | **8.1%** | 4.86 (hits cap) | 33.5% |
| Reader (20 s) | **46.0%** | 0 | n/a |
| Mash-random (50 ms, random) | 4.7% | 4.83 | 32.2% |

- **~38 percentage-point** win-rate penalty for mashing.
- Reader never trips a single emergency.
- Correct-pick rate ≈ 1/3 = pure random — confirms variant + shuffle leave a masher no memorizable pattern.
