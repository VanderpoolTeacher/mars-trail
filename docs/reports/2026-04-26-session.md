## v0.11.0 — The Lounge

A dedicated soundtrack jukebox screen reachable from a floating vinyl button in the lower-right.

- **Vinyl button** (`#lounge-open`, `.lounge-vinyl`): always-visible CSS-only record icon with theme-colored center label and an asymmetric outer arc that spins via `animation: lounge-vinyl-spin` on hover. Single entry point everywhere — z-index 550 sits above the title-layer (500) and below the Lounge layer (600) and modal-root (1000).
- **Lounge layer** (`src/ui/lounge.js`, `#lounge-layer`): full-screen overlay with header (title + theme select + close), now-playing card, and a scrollable list of all 8 tracks. ESC or `CLOSE ✕` exits; music keeps playing seamlessly across open/close.
- **Per-theme flavor copy** (`src/content/trackFlavor.js`): each track has 4 in-universe one-liners — Mission Control field-recording log, LCARS Stardate entry, Starfighter pilot mixtape note, Voltron Paladin recording.
- **In-Lounge theme switcher**: needed because the topbar dropdown is covered while the Lounge is open. Triggers a re-render of flavor copy and skin without leaving.
- **Audio API extensions** (`src/audio.js`): `seekTo(fraction)`, `getProgress()`, `getDuration()`, `getCurrentTime()`, `isPlaying()`, `togglePlayPause()`, `pauseAudio()`, `resumeAudio()`, `getCurrentTrackId()`, `onTimeUpdate(cb)`. Also dropped the title-screen guard from `selectTrack` so picking a row from the Lounge during the title screen actually plays the track.
- **Per-theme Lounge skins** added to each `theme-*.css` using the existing `body[data-theme="..."]` selector pattern.

## v0.12.0 — Soundtrack visualizer + bubble-pop game

A 16:9 reactive panel beside the now-playing card. Started as a "visuals for the songs" brainstorm, ended as a small pop-the-bubbles game.

### Engine (`src/ui/visualizer.js`)

Single `<canvas>` driven by `requestAnimationFrame`. Layers, drawn back-to-front each frame:

1. Solid background fill in palette `bg`
2. Slow rotating conic gradient (~40s/rev) in palette `accent` — gives the panel a "drifting through space" feel
3. **22 ambient orbs** with varied size (10–40% of min-dim) and varied per-orb intensity (alpha multiplier 0.35–1.20). Independent low-frequency sine drifters, no two synced
4. Static halo orb anchored at panel center (diamond color)
5. **Diamond hazard** — yellow rotating diamond (`drawDiamond`)
6. **Bubbles** (see below)
7. **Pulse rings** spawned on amplitude transients
8. HUD via HTML overlay (not canvas — see below)

### Audio pipe (`src/audio.js`)

`getAnalyser()` lazily creates an `AudioContext` + `MediaElementAudioSourceNode` from the existing music `Audio` element + an `AnalyserNode` (`fftSize=256`, `smoothing=0.6`). Idempotent. Wires `source → analyser → destination` so audio still plays through the speakers. `resumeAudioContext()` is called on visualizer start in case the browser auto-suspended on tab blur.

Pulse-ring trigger: RMS over time-domain data must exceed a 30-frame moving average by `TRIGGER_FACTOR=1.40` AND at least `TRIGGER_MIN_GAP=350ms` since the last spawn. Cap of 8 concurrent rings. Each ring fades-out as it expands over 900ms.

### Per-track palettes (`src/content/trackPalettes.js`)

Engine ignores the active theme — palette is per-track.

### Bubble physics

Pixel-space `{ x, y, vx, vy, r }` integrated each frame with a capped `dt` (max 48ms to avoid tab-blur fling). On canvas resize (fullscreen toggle, window resize) positions/velocities/radii are scaled proportionally.

- **Side-entry**: bubbles spawn just past one of the four edges with an inward velocity at `ENTRY_SPEED=0.16 px/ms`. Velocity magnitude decays via easeOutCubic to a per-bubble `driftSpeed` (0.012–0.034 px/ms) over `ENTRY_DURATION=5500ms`. Speed is then renormalized each frame so collisions only redirect — a neat side-effect: motion stays calm regardless of bumping.
- **Fade-in**: bubble alpha ramps 0→1 over the first 1.5s after spawn.
- **Wall bounces** activate only after the bubble first enters the panel (`b.inside` flag), so it doesn't bounce off the wall it spawned through.
- **Pairwise elastic collisions** (equal mass): swap velocity components along the collision normal, de-overlap by half each.
- **Highlight**: clear (outlined) ellipse that orbits the bubble center and rotates on its own axis. Per-bubble random direction (sign), speed, and size. Highlight alpha is dimmed by bubble luminance (`0.35 * Math.min(1, 0.15 + lum * 1.5)`) so dark-palette bubbles aren't visually punched by a bright white ring.

### The game

- **Click a bubble** → `popBubble(b, 'good')` — score +1, `playSfx('good')` (triangle wave 660→1320 Hz, 220ms decay)
- **Bubble hits the diamond** → `popBubble(b, 'bad')` — score −1, `playSfx('bad')` (square wave 220→70 Hz, 300ms decay)
- Both pops: body fades fast (~25% of pop duration), then 14 fragments fly outward at varied speeds for `POP_DURATION=1800ms`, traveling up to `Math.max(w,h) * 1.1` so they fly past every edge before respawning the bubble from a fresh side
- **SFX synthesized on demand** — `OscillatorNode` + `GainNode`, no audio assets
- **SFX toggle** in the visualizer footer (🔊/🔇), persisted via `marsTrail.sfxMute`, separate from music mute

### HUD (HTML overlay inside the visualizer frame)

```
{TRACK NAME}
{elapsed} / {duration}
SCORE +N    BUBBLES N
```

Lives inside `.lounge-visualizer-frame` so it goes fullscreen with the canvas. Updated by a 200ms `setInterval` while the Lounge is open. `pointer-events: none` so clicks pass through to the bubble-pop handler.

### Controls

- **Click** → pop bubble
- **F** → toggle fullscreen on the visualizer frame (works whether in or out of fullscreen)
- **ESC** → exit fullscreen first; close Lounge if not in fullscreen
- **↑ / ↓** (Lounge open) → add / remove bubbles, clamped 1–14
- **← / →** → cycle music tracks (was ↑ / ↓ pre-v0.12.0; remapped because the Lounge needed up/down)
- The narrow-viewport CSS rule re-orders the visualizer above the now-playing card on `<= 640px`.

## v0.12.1 — Mobile landscape auto-fullscreen + score-only HUD (#74)

On a phone-sized viewport in landscape with the Lounge open, the visualizer auto-fullscreens and the HUD trims to just the score.

- **Trigger**: `window.matchMedia('(orientation: landscape) and (max-width: 900px)')`. Listener registered on `openLounge`, removed on `close`.
- **CSS** (`styles/components.css`): same media query hides `.hud-track`, `.hud-time`, and `#hud-bubbles`. Independent of fullscreen so it still applies if `requestFullscreen` is denied (iOS Safari without a user gesture).
- **JS**: on `matches=true` calls `requestFullscreen()` on the frame; on `matches=false` calls `exitFullscreen()`. Both wrapped in try/catch with promise-rejection swallowing.
- **Desktop unaffected** — the `max-width: 900px` filter gates it.

## v0.12.2 — Double-tap visualizer to toggle fullscreen (#76, no issue)

Same shared `toggleVisualizerFullscreen()` helper used by the F key. Click handler tracks `lastClickAt`; if a second click arrives within 300ms it triggers fullscreen instead of popping a bubble. The first tap still pops as normal — the trade-off is "you might pop a bubble on the way to fullscreen", which is fine for the game's responsiveness. Hint label updated to `POP BUBBLES BEFORE THEY HIT THE DIAMOND · F OR DOUBLE-TAP FULLSCREEN`.

## Notable decisions / gotchas

- **Z-index ordering** matters more than usual: title-layer (500) ➜ Lounge layer (600) ➜ modal-root (1000); floating vinyl button at 550 sits above title but below the Lounge so it's covered when the Lounge is open. First Lounge attempt at z-index 90 silently rendered behind the title screen.
- **`createMediaElementSource` is one-shot per `<audio>` element**: `getAnalyser()` is wrapped in try/catch and is idempotent so re-opening the Lounge doesn't throw.
- **`pointer-events: none` on the canvas** lets the parent frame receive clicks without the canvas swallowing them. Same for `.lounge-hud`.
- **Fullscreen API is split-brain across browsers**: `requestFullscreen` / `webkitRequestFullscreen`, `document.fullscreenElement` / `document.webkitFullscreenElement`, `exitFullscreen` / `webkitExitFullscreen`. Vendor-prefixed forms still needed for Safari.
- **`matchMedia` event API** is also split: modern browsers expose `addEventListener('change', cb)`; older Safari only has `addListener(cb)`. Both code paths registered in `openLounge`.
- **iOS Safari can deny programmatic fullscreen** outside of a direct user gesture. The score-only HUD is CSS-driven on the same media query precisely so the experience still degrades cleanly in that case.
